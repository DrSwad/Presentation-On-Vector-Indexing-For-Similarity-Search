Helpful Libraries

FAISS

  • By Meta
  • Rich set of index types
  • Primarily in-memory
  • Ideal for solutions involving multiple index types and fitting in memory

ScaNN

  • By Google
  • Optimized for MIPS and superior recall
  • Less flexibility in index types and primarily focused on dot-product similarity
  • Ideal for transformer-based models and high recall requirements

DiskANN

  • By Microsoft
  • Disk-based vector search over dataset that doesn't fit in memory
  • Higher latency than in-memory solutions
  • Ideal for large datasets where memory costs are a concern
